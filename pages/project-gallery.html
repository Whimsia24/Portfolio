Project Case Study (PAR Model)

Online Art Gallery Website

Live Site: whimsia24.github.io/gallery

P — Problem

This project was developed as a class assignment with the goal of creating a clean, visual web experience to present gallery-style content in an engaging and easy-to-browse format. The challenge was to design and implement a responsive layout that maintains visual appeal and usability across different screen sizes while showcasing content effectively.

A — Action

I designed and built a responsive gallery website using HTML for structure, CSS for styling and layout, and JavaScript for interactivity. I structured the content for readability and visual hierarchy, implemented responsive design techniques to work on desktop and mobile devices, and applied consistent styling throughout. The project was then published using GitHub Pages for live hosting and easy sharing.

R — Result

The completed gallery website demonstrates a polished front-end design that showcases content clearly and attractively. This project strengthened my skills in responsive layout design, CSS styling consistency, and deployment workflows, while providing a live, portfolio-quality web sample that employers can review.

Project Structure Outline

Overview: Responsive gallery website with clean front-end layout and GitHub Pages deployment.

Features:

  • Responsive design for desktop and mobile
  • Clean visual hierarchy and content layout
  • Consistent styling and typography
  • Live deployment via GitHub Pages

Technologies Used: HTML, CSS, JavaScript, GitHub Pages

Challenges and Takeaways: Responsive grid design, cross-browser compatibility, and maintaining visual consistency across resolutions.
