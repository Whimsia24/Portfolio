Project Case Study (PAR Model)

Character Tracker

Live Demo: vtravis-fullstack.onrender.com

P — Problem

This application was developed primarily as a class project for personal use, with the goal of creating a centralized, user-friendly system to track and manage character information in real time. The project focused on full-stack development concepts, including server-side logic, database integration, and front-end interaction. The core challenge was delivering a complete application that handled user input and persistent character data efficiently.

A — Action

I developed a full-stack character tracker using a modern JavaScript stack and implemented both front-end and back-end functionality. The user interface was designed to be responsive and intuitive across screen sizes. On the server side, I implemented RESTful routes for create, read, update, and delete operations, and connected the app to a database for persistent storage. I deployed the application on Render so it could be tested in a live environment.

R — Result

The completed app allows users to create, view, update, and delete character records through a responsive interface. This project strengthened my understanding of full-stack architecture, client-server communication, and deployment workflows, while improving my ability to debug issues in a production environment.

Project Structure Outline

Overview: Full-stack character tracker with a responsive front end, server-side routes, and persistent database storage.

Features:

  • Create, read, update, and delete data
  • Server-side routing and database integration
  • Responsive layout for desktop and mobile
  • Live deployment on Render

Technologies Used: HTML, CSS, JavaScript, Node.js, Express, database integration, Render deployment

Challenges and Takeaways: Understanding full-stack data flow, handling deployment and environment configuration, and improving maintainability.
